These rules are ALWAYS ACTIVE for all files matching `packages/core/**`.
</rule_activation>

### Rules

- **R-HOOK-001** MUST: Implement custom React hooks as the primary pattern for encapsulating and sharing reusable UI logic throughout the core library.
- **R-HOOK-002** MUST: Follow the `use-*` naming convention for all custom hooks (e.g., use-local-value, use-is-focused, use-deep-field).
- **R-HOOK-003** MUST: Isolate each hook in its own module within the lib directory or component-specific lib subdirectories.
- **R-HOOK-004** MUST: Follow the single-responsibility principle - each hook should encapsulate one specific concern or behavior.
- **R-HOOK-005** MUST: Encapsulate state management, side effects, context consumption, and complex behavioral logic within hooks.
- **R-HOOK-006** MUST: Expose clean, well-defined interfaces to consuming components.
- **R-HOOK-007** SHOULD: Separate stateful logic from presentational components to maintain separation of concerns.
- **R-HOOK-008** SHOULD: Design hooks to be independently testable without requiring full component rendering.
- **R-HOOK-009** SHOULD: Compose complex behaviors by combining multiple focused hooks rather than creating monolithic hooks.
- **R-HOOK-010** MAY: Create specialized hooks for specific concerns such as drag-and-drop functionality, preview modes, canvas rendering, and field management.
- **R-HOOK-011** MUST NOT: Create hooks prematurely before patterns are fully understood and validated through usage.
- **R-HOOK-012** MUST NOT: Violate React hooks lifecycle rules (e.g., calling hooks conditionally or in loops).

```bash
# Verify hooks are isolated in lib directories
find packages/core -path "*/lib/use-*.ts" -o -path "*/lib/use-*.tsx"

# Check for hooks violating Rules of Hooks (basic pattern check)
find packages/core -name "use-*.ts" -o -name "use-*.tsx" | xargs grep -E "if.*use[A-Z]|for.*use[A-Z]|while.*use[A-Z]"
```

**Accept when:**
- All custom hooks follow the `use-*` naming convention
- Each hook is isolated in its own module within a lib directory
- Hooks encapsulate single, well-defined concerns (no monolithic hooks)
- Hooks do not violate React hooks rules (no conditional or loop-based hook calls)
- Stateful logic is separated from presentational components
- Hook interfaces are clean and expose only necessary functionality
- Complex behaviors are composed from multiple focused hooks rather than duplicating logic
