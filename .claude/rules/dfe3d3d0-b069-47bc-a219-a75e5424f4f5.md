### Rules

- **R-SNAP-001** MUST: Use snapshot testing for core data transformation functions (walk-tree, walk-app-state) that produce structured outputs involving nested data structures, tree traversals, or complex state mutations.
- **R-SNAP-002** MUST: Use snapshot testing for reducer actions (such as remove) that perform state management operations.
- **R-SNAP-003** MUST: Co-locate test files in `__tests__` directories alongside the source code (e.g., `packages/core/reducer/actions/__tests__/`, `packages/core/lib/data/__tests__/`).
- **R-SNAP-004** MUST: Version-control all snapshot files alongside the code to enable change detection in CI/CD pipeline.
- **R-SNAP-005** MUST: Apply snapshot testing to the testing.snapshot facet within the CI/CD pipeline to ensure output consistency across builds.
- **R-SNAP-006** SHOULD: Carefully review snapshot diffs before updating to avoid masking bugs through lazy updates.
- **R-SNAP-007** SHOULD: Use snapshot testing to validate entire output structures in a single test rather than verbose assertion chains for complex nested objects.
- **R-SNAP-008** MAY: Update snapshots in bulk when intentional changes occur, but only after verifying the changes are correct.

```bash
# Verify snapshot tests exist for core transformation functions
find packages/core/lib/data/__tests__ -name '*.test.*' -o -name '*.spec.*' | xargs grep -l 'toMatchSnapshot\|toMatchInlineSnapshot' || echo "WARNING: No snapshot tests found for data transformations"

# Verify snapshot tests exist for reducer actions
find packages/core/reducer/actions/__tests__ -name '*.test.*' -o -name '*.spec.*' | xargs grep -l 'toMatchSnapshot\|toMatchInlineSnapshot' || echo "WARNING: No snapshot tests found for reducer actions"

# Verify snapshot files are tracked in git
find packages/core -name '__snapshots__' -type d | while read dir; do
  if [ -d "$dir" ]; then
    git ls-files "$dir" | grep -q '.' || echo "WARNING: Snapshot directory $dir not tracked in git"
  fi
done

# Verify __tests__ directory structure
find packages/core/lib/data packages/core/reducer/actions -name '__tests__' -type d || echo "WARNING: Missing __tests__ directories"
```

- Snapshot tests exist for core data transformation functions (walk-tree, walk-app-state)
- Snapshot tests exist for reducer actions performing state management operations
- Test files are organized in `__tests__` directories co-located with source code
- Snapshot files are version-controlled in git
- CI/CD pipeline includes testing.snapshot facet validation
