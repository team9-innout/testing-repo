<rule_activation id="0f712a72-1d1c-44d5-a14c-cf29ec7a45ea" title="Comprehensive Unit Testing for Core Reducer Actions" applies_to="packages/core/reducer/actions/**">
These rules are ALWAYS ACTIVE for all files matching `packages/core/reducer/actions/**` to ensure comprehensive unit testing of core state management logic.
</rule_activation>

### Rules

- **R-UT-001** MUST: Each core reducer action (duplicate, move, set-ui, replace, insert) MUST have a dedicated unit test file in `packages/core/reducer/actions/__tests__/` directory.
- **R-UT-002** MUST: Test files MUST follow the naming convention `[action-name].spec.ts` where action-name corresponds to the reducer action being tested.
- **R-UT-003** MUST: Each action test file MUST have isolated test specifications with clear separation of concerns.
- **R-UT-004** SHOULD: Test coverage for core reducer actions SHOULD maintain at least 91% significance to ensure high confidence in state management logic.
- **R-UT-005** MUST: Tests MUST cover both positive scenarios (expected state transformations) and negative scenarios (edge cases, error conditions).
- **R-UT-006** SHOULD: Tests SHOULD serve as living documentation for expected behavior of each action, with descriptive test names and clear assertions.
- **R-UT-007** MUST: When modifying a reducer action, corresponding tests MUST be updated to maintain synchronization between implementation and test coverage.
- **R-UT-008** SHOULD: Tests SHOULD be structured to enable parallel test execution for faster CI/CD pipelines.

### Verify

```bash
# Verify all core actions have corresponding test files
for action in duplicate move set-ui replace insert; do
  if [ ! -f "packages/core/reducer/actions/__tests__/${action}.spec.ts" ]; then
    echo "ERROR: Missing test file for ${action} action"
    exit 1
  fi
done

# Verify test naming convention
find packages/core/reducer/actions/__tests__ -name '*.spec.ts' | while read -r file; do
  basename "$file" | grep -qE '^[a-z-]+\.spec\.ts$' || {
    echo "ERROR: Test file $(basename "$file") does not follow naming convention [action-name].spec.ts"
    exit 1
  }
done

# Run tests for core reducer actions
npm test -- packages/core/reducer/actions/__tests__/ || {
  echo "ERROR: Core reducer action tests failed"
  exit 1
}

# Verify test coverage (if coverage tooling is available)
if command -v npm &> /dev/null; then
  npm run test:coverage -- packages/core/reducer/actions/ 2>/dev/null || true
fi
```

**Accept when:**
- All core reducer actions (duplicate, move, set-ui, replace, insert) have corresponding test files in `packages/core/reducer/actions/__tests__/`
- All test files follow the `[action-name].spec.ts` naming convention
- All tests pass successfully without errors
- Test coverage for core reducer actions maintains at least 91% significance
- Tests are isolated and can be executed in parallel
- Tests clearly document expected behavior with descriptive test names
- Modified actions have synchronized test updates
