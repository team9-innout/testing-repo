### Rules

- **R-TEST-001** MUST: Test files must be placed in `__tests__` directories adjacent to the code they test (co-located pattern).
- **R-TEST-002** MUST: Test files must use the `.spec.tsx` extension for TypeScript/TSX test files that include React component testing with JSX syntax.
- **R-TEST-003** SHOULD: Test file names should mirror the tested module names for easy identification and maintenance (e.g., `resolve-component-data.spec.tsx` tests `resolve-component-data.ts`).
- **R-TEST-004** MUST: Apply the `__tests__` directory pattern consistently across both component directories (e.g., `components/Puck/__tests__/`) and library function directories (e.g., `lib/__tests__/`).
- **R-TEST-005** SHOULD: Avoid mixing test file patterns - do not place `.test.tsx` files alongside `__tests__` directories without clear guidelines to prevent pattern duplication.

### Context

This pattern emerged to address the need for:
- Discoverable and maintainable tests alongside the code they test
- TypeScript/TSX support for React component testing
- Clear separation between test code and production code
- Easy test discovery by testing frameworks (Jest, Vitest)
- Scalability across monorepo structures with multiple packages

Key benefits:
- Co-located tests are easy to find and maintain
- `__tests__` directories are automatically recognized by testing frameworks
- Clear separation through dedicated directories
- `.spec.tsx` extension clearly identifies test files
- Easy exclusion from production builds via glob patterns

Tradeoffs:
- Additional directory nesting increases path length
- Requires discipline to maintain consistency

### Verify

```bash
# Check that test files exist in __tests__ directories
find packages/core -type f -name "*.spec.tsx" | grep -v "__tests__" && echo "ERROR: Test files found outside __tests__ directories" || echo "PASS: All .spec.tsx files in __tests__ directories"

# Verify test files use .spec.tsx extension within __tests__ directories
find packages/core -type d -name "__tests__" -exec find {} -type f ! -name "*.spec.tsx" ! -name "*.spec.ts" \; | grep -E "\.(tsx?|jsx?)$" && echo "ERROR: Non-spec test files found in __tests__" || echo "PASS: All test files use .spec.tsx/.spec.ts extension"

# Check for mixed patterns (both .test.tsx and __tests__ directories in same location)
for dir in $(find packages/core -type d -name "__tests__" -exec dirname {} \;); do
  if find "$dir" -maxdepth 1 -name "*.test.tsx" -o -name "*.test.ts" | grep -q .; then
    echo "WARNING: Mixed test patterns in $dir"
  fi
done
```

**Accept when:**
- All test files in packages/core are located within `__tests__` directories
- Test files use the `.spec.tsx` (or `.spec.ts`) extension
- Test files are co-located adjacent to the code they test
- Pattern is applied consistently across components/ and lib/ directories
- No `.test.tsx` files exist in directories that also contain `__tests__` directories
- Test directory structure mirrors production code organization
