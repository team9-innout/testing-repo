<rule_activation id="80aadadd-3555-46e7-b9f4-8836f0decd06" title="Adopt Index File Pattern for Component Module Exports in UI Framework" applies_to="packages/core/components/**">
These rules are ALWAYS ACTIVE for all files matching `packages/core/components/**`.
</rule_activation>

### Rules

- **R-INDEX-001** MUST: Each component directory MUST contain an `index.tsx` file that serves as the single entry point for that component.
- **R-INDEX-002** MUST: The `index.tsx` file MUST re-export the component's public API and hide internal implementation details.
- **R-INDEX-003** MUST: Component imports MUST use the directory path (e.g., `'components/Canvas'`) rather than direct file paths (e.g., `'components/Canvas/Canvas.tsx'`).
- **R-INDEX-004** MUST: Internal implementation files within a component directory MUST NOT be imported directly from outside that component directory.
- **R-INDEX-005** MUST: This pattern MUST be consistently applied across core components including Canvas, Layout, and ViewportControls within the Puck component system.
- **R-INDEX-006** SHOULD: When creating new components, apply this index file pattern to maintain consistency across the codebase.
- **R-INDEX-007** SHOULD: Carefully manage dependencies to avoid circular dependency issues when using the index file pattern.
- **R-INDEX-008** MAY: Use the index file to re-export types, utilities, and subcomponents that are part of the component's public API.

### Verify

**Note**: The source ADR does not include explicit verification commands. Consider implementing checks such as:

```bash
# Check that each component directory has an index.tsx
find packages/core/components -mindepth 1 -maxdepth 1 -type d | while read dir; do
  if [ ! -f "$dir/index.tsx" ]; then
    echo "Missing index.tsx in $dir"
  fi
done

# Check for direct imports of component internals (excluding index imports)
# This would require static analysis of import statements
grep -r "from.*components/.*/.*\.tsx" packages/ --include="*.ts" --include="*.tsx" | \
  grep -v "from.*components/[^/]*/index" || true
```

**Accept when:**
- Each component directory under `packages/core/components/` contains an `index.tsx` file
- The `index.tsx` file exports the component's public API
- Component imports use directory paths (e.g., `'components/Canvas'`) rather than specific file paths
- No external imports directly reference internal component files (only the index)
- The pattern is consistently applied to Canvas, Layout, ViewportControls, and other core components
