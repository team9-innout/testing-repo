### Rules

- **R-TEST-001** MUST: Place unit test files in `__tests__` subdirectories adjacent to the code they test.
- **R-TEST-002** MUST: Use the `.spec.tsx` extension for React component tests.
- **R-TEST-003** MUST: Use the `.spec.tsx` or `.spec.ts` extension for TypeScript test files.
- **R-TEST-004** MUST: Apply the `__tests__` convention consistently across all architectural layers (UI components, library utilities, and internal helpers).
- **R-TEST-005** SHOULD: Follow Jest's default test discovery patterns when organizing tests.
- **R-TEST-006** SHOULD: Follow React Testing Library practices for component testing.
- **R-TEST-007** MUST: Ensure test files remain co-located with implementation code - when refactoring or moving code, move associated tests together.

### Verify

**Note**: The source ADR does not contain an explicit verify section. The following verification commands check for compliance with the __tests__ directory convention:

```bash
# Check that test files are in __tests__ directories
find packages/core -name '*.spec.ts*' -not -path '*/__tests__/*' || echo "All test files properly located in __tests__ directories"

# Verify React component tests use .spec.tsx extension
find packages/core -path '*/__tests__/*.tsx' -not -name '*.spec.tsx' && echo "Found component test files not using .spec.tsx extension" || echo "All component tests use correct extension"

# Confirm __tests__ directories exist adjacent to source files
find packages/core -type f -name '*.spec.ts*' -exec dirname {} \; | sort -u | xargs -I {} test -d "$(dirname {})/.." && echo "Tests co-located with source code"
```

**Accept when:**
- All test files for components, utilities, and reducers are located in `__tests__` subdirectories adjacent to their source code
- Test files use `.spec.tsx` for React components and `.spec.ts` or `.spec.tsx` for TypeScript files
- Test organization is consistent across UI components (like Puck), library utilities (like resolve-component-data, migrate), and internal helpers (like reducer actions)
- Tests can be discovered by Jest's default patterns without additional configuration
- When code is moved or refactored, associated tests in `__tests__` directories are moved together
- Build tools are configured to exclude `__tests__` directories from production bundles
