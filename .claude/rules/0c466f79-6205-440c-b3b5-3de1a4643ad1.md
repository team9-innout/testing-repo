### Rules

- **R-MOCK-001** MUST: Mock all external dependencies (APIs, services, external modules) in state management store slice tests to enable isolated unit testing without requiring external service setup.
- **R-MOCK-002** MUST: Mock inter-slice dependencies when testing individual store slices to ensure test isolation and prevent cascading failures across store components.
- **R-MOCK-003** MUST: Apply mocking patterns consistently across all store slice tests (nodes, fields, history, permissions) to maintain a standardized testing approach.
- **R-MOCK-004** SHOULD: Design tests to run in parallel without resource contention or state pollution by ensuring complete isolation through mocking.
- **R-MOCK-005** SHOULD: Ensure mocked dependencies accurately represent real implementation behavior, including edge cases and error conditions, to prevent false confidence in test results.
- **R-MOCK-006** SHOULD: Keep mock implementations synchronized with production code changes to prevent divergence between test and production behavior.
- **R-MOCK-007** MAY: Document mock behavior and assumptions in test files to aid maintainability and developer onboarding.
